# Backend Java Standards

## Language Version
All backend code targets **Java 25**. Use Java 25-compatible language features and APIs.

## Records for DTOs
Request and response DTOs must be implemented as **Java records**, not classes. Records eliminate boilerplate and enforce immutability for data transfer objects.

**Preferred:**
```java
public record AnimalSearchRequest(
  String query,
  List<DietType> diets,
  List<HabitatType> habitats
) {}
```
**Avoid:** Creating DTO classes with constructors, getters, `equals`, `hashCode`, and `toString` manually.

## No Lombok
Do not add Lombok to the project. Use Java records for DTOs and explicit constructors/getters for JPA entities. Lombok is intentionally absent to keep the code idiomatic with modern Java.

## UUID Primary Keys
Entity primary keys use `UUID` generated by the persistence layer. Do not use sequential numeric IDs.

## Modern Java Patterns
Prefer modern Java idioms:
- Use `instanceof` pattern matching rather than explicit casts
- Use `sealed` classes or records for closed type hierarchies where appropriate
- Avoid raw types; always parameterize generics

## File Naming
Java source files must use **PascalCase** matching the public class, interface, record, or enum they contain: `AnimalController.java`, `ApiResponse.java`, `ErrorCode.java`.

## Package Imports
Always use fully qualified package imports (`com.animals.*`, `org.springframework.*`). Wildcard imports (`import com.animals.catalog.*;`) are not allowed.

**Preferred:** `import com.animals.catalog.animal.service.AnimalCatalogService;`
